# Skills v2 Probe 安全规则

这份文档定义 Probe 阶段允许与禁止的命令边界。

---

## 1. Probe 阶段允许的动作

Probe 阶段允许：

1. 查询系统信息
2. 查询工具是否存在
3. 查询工具版本
4. 查询容器与 shell 能力
5. 查询 GPU 能力

这些动作应尽量是：

1. 只读
2. 无副作用
3. 可重复执行

---

## 2. Probe 阶段禁止的动作

Probe 阶段不应默认执行：

1. `apt install`
2. `yum install`
3. `pip install`
4. `npm install`
5. `conda create`
6. `docker pull`
7. `docker compose up`
8. 修改系统配置
9. 删除文件
10. 重启服务
11. 自动提权
12. 修改 sudoers / 用户组

这些属于部署执行阶段，不属于 Probe 阶段。

---

## 3. Probe 阶段的失败处理

Probe 命令失败时：

1. 记录工具不存在或不可用
2. 不立即转成部署失败
3. 将失败事实交给 Agent 做后续判断

例如：

1. `conda --version` 失败，不代表任务失败，只代表 conda 不可用
2. `docker info` 失败，不代表项目失败，只代表容器能力受限

---

## 4. 什么时候可以升级到执行阶段

只有在 Probe 已经完成、且主 Agent 已经形成部署策略后，才应进入：

1. 依赖安装
2. 镜像构建
3. 容器启动
4. 健康检查

---

## 5. 核心原则

Probe 负责回答：

1. 主机有什么
2. 主机缺什么
3. 哪些路径明显不可用

Probe 不负责回答：

1. 下一步一定执行什么
2. 自动帮用户补环境
3. 自动修复部署问题
4. 自动提升执行身份
